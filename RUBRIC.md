# Capstone Rubric — Definition of Done

This is the bar your repo must clear to "pass" the capstone. Each row is an
**objective, reproducible signal** — a command anyone can run against your repo
to confirm the capability is real, not claimed. The same signals appear as the
**Checkpoint** at the end of each module.

Use this as a self-assessment checklist and as the grading sheet if you're in a
cohort. `<kit>` = your `platform-starter-kit` clone; run commands from your
repo root unless noted.

## Core (required — solo or team)

| # | Capability | Signal (how it's verified) | Module |
|---|------------|----------------------------|--------|
| 1 | **Scaffolded from the kit** | `PLATFORM-KIT.md` exists and records a kit commit SHA | [01](modules/01-scaffold.md) |
| 2 | **Containerized** | `docker compose up -d` boots; a `Dockerfile` exists | [02](modules/02-build-app.md) |
| 3 | **Health/readiness** | `curl -sf localhost:8000/health` and `/ready` both return 200 | [02](modules/02-build-app.md) |
| 4 | **Tests exist & pass** | the project's test command exits 0 (e.g. `pytest`, `npm test`) | [02](modules/02-build-app.md) |
| 5 | **Readiness gate green** | `python3 <kit>/tools/doctor.py .` reports **0 FAIL** | [02](modules/02-build-app.md) |
| 6 | **Shift-left hooks** | `pre-commit run --all-files` passes; `.secrets.baseline` present | [03](modules/03-shiftleft-ci.md) |
| 7 | **CI green** | the GitHub Actions CI run on your branch is green | [03](modules/03-shiftleft-ci.md) |
| 8 | **Observability lit** | a Grafana panel shows request traffic generated by *your* app; one trace is visible in Jaeger | [04](modules/04-observability.md) |
| 9 | **An SLO defined** | `operations/SLOs.md` has at least one SLI/SLO tied to a recording rule | [04](modules/04-observability.md) |
| 10 | **Load test passes** | `k6 run` (or Locust) against your routes meets its declared thresholds | [05](modules/05-load-testing.md) |
| 11 | **Security review done** | `SECURITY-FINDINGS.md` records findings + remediations (incl. authn/z and any payment flow) | [06](modules/06-security-pentest.md) |
| 12 | **IaC plans clean** | `terraform validate` passes and `terraform plan` runs without error | [07](modules/07-iac-governance.md) |
| 13 | **Policy gate understood** | `conftest test` exits 0 on the passing fixture and 1 on the failing one | [07](modules/07-iac-governance.md) |
| 14 | **Migration safety wired** | `check_migrations.py` blocks a deliberately unsafe migration; you applied the expand/contract fix | [07](modules/07-iac-governance.md) |
| 15 | **Day-2 readiness** | a runbook adapted to your service; a release decoupled from deploy via a feature flag | [08](modules/08-day2-ops.md) |
| 16 | **Stack writeup** | `STACK-CHOICES.md` documents your stack and **one swap you evaluated** (cite the kit's `TECH-STACK-SWAP-GUIDE.md`) | all |
| 17 | **In sync & submitted** | `sync_check.py` output captured; completion report maps every checkpoint | [10](modules/10-sync-and-submit.md) |

## Team Track (required for teams; optional for solo)

| # | Capability | Signal | Module |
|---|------------|--------|--------|
| T1 | **Code ownership** | `.github/CODEOWNERS` resolved to real handles; branch protection requires review | [09](modules/09-team-track.md) |
| T2 | **No racing deploys** | `publish.yml` has a `concurrency:` group | [09](modules/09-team-track.md) |
| T3 | **Shared infra state** | Terraform uses a remote backend with locking (not a local `tfstate`) | [09](modules/09-team-track.md) |
| T4 | **Environment parity** | `.devcontainer/` + `.tool-versions` committed; every member builds identically | [09](modules/09-team-track.md) |

## Stretch (optional, for distinction)

- **Admin role + product CRUD** with role-based authorization (gives the
  security review real authz boundaries to probe).
- **Stripe webhooks** with signature verification + idempotency (Module 06
  security review of the handler).
- **Feature-flag-gated checkout** with a documented rollout/rollback story
  (Module 08).
- **Drift detection** wired (`drift-detection.yml` un-gated against a real
  remote state).

> Grading suggestion for cohorts: Core = pass/fail (all 17 required). Team Track
> adds 4 for groups. Stretch items are how you separate a *good* capstone from
> an *excellent* one.
